# 🎁 GiftBasket Manager — 伴手礼管理后台

伴手礼线上店综合管理后台，集商品库存管理、成品方案设计、竞品分析、定价计算与运营任务追踪于一体。

---

## 技术栈

| 层级   | 技术                        |
| ------ | --------------------------- |
| 后端   | Node.js + Express 4.21      |
| 数据库 | better-sqlite3（SQLite，WAL 模式） |
| 前端   | 单页 HTML + CSS + JS（无框架依赖）  |
| 运行环境 | Node.js 18+                 |

---

## 快速启动

### 1. 安装依赖

```bash
cd gift-basket-manager
npm install
```

### 2. 启动服务器

```bash
npm start
# 或
node server.js
```

### 3. 打开浏览器

访问 **http://localhost:3000**

> 数据库文件 `inventory.db` 会在首次启动时自动创建并填充种子数据（71 个商品、3 个成品方案、7 个竞品方案、5 个待办任务）。如果数据库已存在则跳过种子写入。

## 功能概览

### 📊 仪表盘

全局数据概览，包括：

- 商品总数（按分类统计：盒子/辅料/糖果/单品）
- 成品方案总数
- 竞品方案总数
- 库存采购总成本估算
- 待办任务数量
- 最近更新的 5 个成品方案

### 📦 货物清单

管理所有商品物料，分类为四种：

| 分类 | 说明 | 种子数据数量 |
|------|------|------------|
| 🎁 盒子 | 礼盒、编织篮、手提包、透明袋、纸袋等 | 22 |
| 🧵 辅料 | 雪梨纸、丝带、拉菲草、卡片、干花等 | 15 |
| 🍬 糖果 | 软糖、棒棒糖、酥糖、瓜子、果冻等 | 13 |
| 🧸 单品 | 保温杯、眼罩、手霜、梳子、茶包等 | 21 |

支持 **增删改查**，可按分类筛选、按名称搜索，查看每个商品的单价、来源、尺寸等信息。

### 🧺 成品方案

组合多个商品形成伴手礼成品方案。功能包括：

- **创建方案**：选择盒子类型 + 添加商品 + 设定利润率（默认 70%）
- **自动计算**：总成本、利润率、毛利额、建议售价
- **方案管理**：编辑基本信息、增删方案内商品、修改利润率后自动重算
- **状态管理**：草稿 / 已发布 / 下架
- 种子数据包含 3 个已发布方案（竹篮伴手礼、PVC透明手提袋、磨砂透明手提袋）

### 🔍 竞品对比

记录和分析竞品伴手礼方案，用于市场对标。功能包括：

- 记录竞品名称、售价、会员价
- 盒子描述、尺寸
- 内部商品明细
- 种子数据包含 7 个竞品方案（护士节、母亲节、女神节等主题）

### 🧮 计算器

快速定价计算工具，用于灵活测算：

- 从商品库中选择商品并设定数量
- 设定目标利润率
- 自动计算总成本、毛利额、建议售价
- 提供多种取整建议（向上取整、向下取整、尾数 .99 定价、整数定价）

### 📋 运营看板

待办任务管理，用于运营排期与协作：

- 记录任务内容、所属平台（小红书、微信、闲鱼、B端等）
- 优先级标记（0 普通 / 1 高 / 2 中）
- 完成状态勾选
- 增删改查
- 种子数据包含 5 个待办任务

## 项目结构

```
gift-basket-manager/
├── server.js             # 后端入口：Express 路由 + SQLite 初始化 + 种子数据
├── package.json          # 项目依赖与脚本
├── inventory.db          # SQLite 数据库文件（自动生成）
├── public/
│   └── index.html        # 前端单页应用（6 Tab 页面）
└── node_modules/         # 依赖包
```

> `public/index.html` 是一个独立的单页 HTML 文件，内嵌了全部样式与脚本，不依赖任何前端框架。

## API 概览

所有 API 返回 JSON 格式：`{ ok: true/false, data: ..., error: "..." }`

### 仪表盘

| 方法   | 路径              | 说明                           |
| ------ | ----------------- | ------------------------------ |
| `GET`  | `/api/dashboard`  | 获取仪表盘统计数据             |

### 货物清单

| 方法     | 路径                  | 说明                       |
| -------- | --------------------- | -------------------------- |
| `GET`    | `/api/products`       | 获取商品列表（支持 ?category & ?search 过滤） |
| `GET`    | `/api/products/:id`   | 获取单个商品详情           |
| `POST`   | `/api/products`       | 创建新商品                 |
| `PUT`    | `/api/products/:id`   | 更新商品信息               |
| `DELETE` | `/api/products/:id`   | 删除商品（被方案引用时禁止删除） |

### 成品方案

| 方法     | 路径                               | 说明                           |
| -------- | ----------------------------------- | ------------------------------ |
| `GET`    | `/api/bundles`                      | 获取所有成品方案（含商品明细） |
| `GET`    | `/api/bundles/:id`                  | 获取单个方案详情               |
| `POST`   | `/api/bundles`                      | 创建方案（自动计算成本&售价）  |
| `PUT`    | `/api/bundles/:id`                  | 更新方案信息或替换商品         |
| `DELETE` | `/api/bundles/:id`                  | 删除方案                       |
| `POST`   | `/api/bundles/:id/items`            | 向已有方案添加商品             |
| `DELETE` | `/api/bundles/:bundleId/items/:itemId` | 从方案中移除商品           |

### 竞品对比

| 方法     | 路径                       | 说明                       |
| -------- | -------------------------- | -------------------------- |
| `GET`    | `/api/competitors`         | 获取所有竞品方案（含明细） |
| `GET`    | `/api/competitors/:id`     | 获取单个竞品详情           |
| `POST`   | `/api/competitors`         | 创建竞品记录               |
| `PUT`    | `/api/competitors/:id`     | 更新竞品信息及内部商品     |
| `DELETE` | `/api/competitors/:id`     | 删除竞品记录               |

### 计算器

| 方法   | 路径               | 说明                           |
| ------ | ------------------ | ------------------------------ |
| `POST` | `/api/calculator`  | 快速定价计算（含取整建议）     |

### 运营看板

| 方法     | 路径                   | 说明               |
| -------- | ---------------------- | ------------------ |
| `GET`    | `/api/checklist`       | 获取所有待办任务   |
| `POST`   | `/api/checklist`       | 创建新任务         |
| `PUT`    | `/api/checklist/:id`   | 更新任务状态或内容 |
| `DELETE` | `/api/checklist/:id`   | 删除任务           |

## 开发说明

- **端口**: 默认监听 `3000`，可在 `server.js` 中修改 `PORT` 常量。
- **数据库**: 使用 SQLite WAL 模式，支持高并发读取。数据库文件位于项目根目录 `inventory.db`。
- **种子数据**: 首次启动时自动写入，已在数据库时不重复写入。如需重置，删除 `inventory.db` 文件后重启即可。
- **删除保护**: 若商品已被成品方案引用，API 会阻止删除，避免数据不一致。
